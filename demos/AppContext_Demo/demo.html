🚀 AppContext.js 功能与操作流程

按照实际使用顺序，了解你可以在页面上做什么

📦 AppContext 提供的核心功能
👤
用户认证

📁
项目管理

✅
待办事项

🔄
实时同步

🔗
快捷链接

1
🚀 打开页面（自动初始化） 自动执行
当你访问应用时，系统会自动完成一系列初始化操作，无需你手动操作。 这个过程会获取你的身份信息，并加载必要的基础数据。
🔧 AppContext 自动调用的接口：
GET /userinfo → 获取你的用户名和部门
GET /template/getUnified → 获取部门列表
GET /template/teamMembers → 获取团队成员信息
GET /template/standardLinks → 获取快捷链接
GET /project/listProjects → 加载你的项目列表
GET /todo/list → 加载你的待办事项
GET /sse/stream → 建立实时推送连接
🖥️ 页面加载完成后你会看到：

📋 我的待办
☐ 完成项目报告
☐ 代码审查
☐ 团队会议
📁 我的项目
🔵 项目 A (Running)
🟡 项目 B (Quotation)
🟢 项目 C (SOP)
2
📁 查看项目列表和统计 用户操作
在页面右侧，你可以看到所有项目的列表和统计信息。 系统会自动统计不同状态的项目数量（Quotation/Running/SOP）。
👆 你可以进行的操作：
  • 点击项目名称查看详情
  • 查看统计图表了解项目分布
  • 拖拽项目调整排序顺序
🔧 涉及的功能方法：
GET /project/getAllProjectTags → 获取项目标签统计
POST /project/reorderProjects → 保存拖拽排序结果
3
➕ 创建新项目 用户操作
当你需要开始一个新项目时，可以创建项目并设置相关信息。 创建后项目会立即显示在列表中。
👆 操作步骤：
  • 点击"新建项目"按钮
  • 填写项目名称
  • 选择所属部门
  • 添加标签（如 Running, Quotation）
  • 填写备注说明
  • 点击保存
🔧 调用的接口：
POST /project/createProject → 创建新项目
💻 AppContext 中的对应代码：
const createProject = (values) => { const payload = { username: userId, projectName: values.projectName, department: values.department, owner: userId, editors: [userId], comment: values.comment, tags: values.tags // ["Running"] 或 ["Quotation"] }; return request("POST", API.BASE + API.PROJECT_CREATE, { data: payload }); };
4
📋 查看和管理待办事项 用户操作
在页面左侧，你可以看到所有待办事项列表。 可以筛选、搜索、排序你的任务。
👆 你可以进行的操作：
  • 在搜索框输入关键词筛选待办
  • 点击待办标题查看详情
  • 勾选完成状态
  • 拖拽调整待办顺序
  • 点击链接打开相关资源
🔧 涉及的功能方法：
POST /todo/update → 更新待办状态/内容
POST /todo/reorder → 保存拖拽排序
POST /open/link (本地) → 打开外部链接
5
✏️ 创建新待办 用户操作
为项目或日常工作创建待办任务，可以设置截止日期、分配给团队成员。
👆 操作步骤：
  • 点击"新建待办"按钮
  • 填写待办标题
  • 选择截止日期
  • 分配给负责人（可选）
  • 添加标签分类
  • 添加相关链接（可选）
  • 填写备注
  • 点击保存
🔧 调用的接口：
POST /todo/create → 创建新待办
💻 AppContext 中的对应代码：
const createTodoV2 = (values) => { const payload = { title: values.title, due_date: values.due_date, // "2024-01-15" comment: values.comment, tags: values.tags, // ["紧急", "重要"] link: values.link, // 相关链接 assignee_ids: values.assignee_ids, // ["zhangsan"] operator_id: userId }; return request("POST", API.BASE + API.TODO_V2.CREATE, { data: payload }); };
6
📝 编辑或删除项目/待办 用户操作
随时修改项目或待办的信息，或删除不需要的条目。
👆 你可以进行的操作：
  • 点击编辑按钮修改项目/待办信息
  • 修改标题、备注、标签等内容
  • 点击删除按钮移除项目/待办
🔧 调用的接口：
POST /project/editProjectMeta → 编辑项目
POST /project/deleteProject → 删除项目
POST /todo/update → 编辑待办
POST /todo/delete → 删除待办
7
🔗 使用快捷链接 用户操作
点击常用系统链接，快速访问其他业务系统。
👆 操作步骤：
  • 在搜索栏下方找到快捷链接区域
  • 点击想要访问的系统链接
  • 系统会通过本地客户端打开链接
🔧 调用的接口：
POST /open/link (本地服务) → 打开外部链接
💡 为什么需要本地服务打开链接？
因为某些内部系统需要在特定浏览器或环境中打开，通过本地客户端（端口7175） 可以确保链接在正确的应用程序中打开，而不是在当前浏览器标签页中。
8
🔄 实时同步（后台自动） 自动执行
当你或其他用户修改数据时，页面会自动刷新显示最新内容，无需手动刷新。
🔧 涉及的功能：
GET /sse/stream → SSE 长连接持续监听
🔄 实时同步场景示例：
10:00 用户A 创建了项目 "新项目X"
10:00 你的页面自动刷新，显示 "新项目X"
10:05 用户B 完成了待办 "代码审查"
10:05 你的页面自动更新，该待办显示为已完成
💡 SSE 是什么？
SSE（Server-Sent Events）是一种服务器向浏览器推送实时数据的技术。 与 WebSocket 不同，SSE 是单向的（服务器→客户端），适合用于接收通知和更新。

📚 总结：AppContext 的作用

AppContext.js 是整个应用的数据中心，它：

  • 管理所有与服务器通信的 API 接口
  • 维护全局状态（用户信息、项目列表、待办事项等）
  • 提供业务方法供组件调用（创建、编辑、删除等）
  • 处理实时数据同步（SSE）
  • 统一管理 API 配置和请求逻辑

所有组件都通过 useContext(AppContext) 来获取数据和调用方法，实现组件间的数据共享和状态管理。
